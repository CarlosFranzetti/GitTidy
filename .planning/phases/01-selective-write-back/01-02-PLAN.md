**Step 1 — Add `confirmApplySelected` function** inside the `App` function body, alongside the other handlers (after `toggleWriteSelection` is a natural location):

```typescript
function confirmApplySelected() {
  if (!activeRepo || !generated || !selectedOwnerRepo || !writeSelection) return

  const selectedItems: string[] = []
  if (writeSelection.readme) selectedItems.push('README rewrite')
  if (writeSelection.description) selectedItems.push('Description')
  if (writeSelection.topics) selectedItems.push('Topics')
  if (writeSelection.homepage) selectedItems.push('Deploy suggestion / homepage')

  if (selectedItems.length === 0) return

  const itemList = selectedItems.map((item) => `• ${item}`).join('\n')
  const modalBody =
    `The following items will be written to ${activeRepo.repo.fullName}:\n\n${itemList}\n\n` +
    'Items you left unchecked will not be changed.'

  setPendingAction({
    title: 'Apply selected improvements?',
    body: modalBody,
    confirmLabel: `Apply ${selectedItems.length} item${selectedItems.length === 1 ? '' : 's'}`,
    run: async () => {
      if (writeSelection.readme) {
        await updateRepositoryReadme({
          owner: selectedOwnerRepo.owner,
          repo: selectedOwnerRepo.name,
          token: githubToken,
          content: generated.readmeMd,
          sha: activeRepo.readmeSha,
        })
      }

      const needsMetadata =
        writeSelection.description ||
        writeSelection.topics ||
        writeSelection.homepage

      if (needsMetadata) {
        await updateRepositoryMetadata({
          owner: selectedOwnerRepo.owner,
          repo: selectedOwnerRepo.name,
          token: githubToken,
          description: writeSelection.description
            ? generated.description
            : activeRepo.repo.description,
          topics: writeSelection.topics
            ? generated.topics
            : activeRepo.repo.topics,
          homepage: writeSelection.homepage
            ? homepageDraft.trim()
            : activeRepo.repo.homepage,
        })
      }

      await selectRepo(activeRepo.repo)
    },
  })
}
```

**Why this implementation is correct:**
- `updateRepositoryMetadata` requires all three fields; unchecked fields fall back to `activeRepo.repo.*` so existing GitHub values are preserved.
- `updateRepositoryReadme` is only called when `writeSelection.readme` is true.
- The modal body explicitly lists every item being written — no surprises.
- `confirmLabel` shows the count ("Apply 2 items") making the intent clear.
- The guard `if (selectedItems.length === 0) return` is defense-in-depth (button is disabled in this state but worth guarding).
- `await selectRepo(activeRepo.repo)` refreshes the active repo after writes complete.
- The `run` function is called by the existing `runPendingAction` handler which already handles `setIsWriting`, `setMessage`, and `setPendingAction(null)` — no changes needed there.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | confirmApplySelected run function | mitigate | Guard checks activeRepo, generated, selectedOwnerRepo, writeSelection are all non-null before constructing the write. If any is missing, returns early. |
| T-02-02 | Elevation of Privilege | metadata fallback to activeRepo.repo.* | accept | Unchecked fields fall back to current GitHub values — worst case is a no-op re-write of existing data, not an escalation. |
| T-02-03 | Denial of Service | rapid multiple confirms | accept | Existing isWriting state disables the Confirm button during write; duplicate submits are already blocked by the existing ConfirmationModal + runPendingAction pattern. |
| T-02-04 | Information Disclosure | modal body lists item names | accept | Item names ("README rewrite", "Description", etc.) are generic labels, not repo data. No PII or secrets are exposed in the modal. |
